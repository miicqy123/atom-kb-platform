# 上传功能实现总结

## 已完成的工作

### 1. 上传功能 UI
- 创建了 UploadDialog 组件，包含：
  - 文件选择器（支持 PDF、Word、Excel、PPT、TXT、MD）
  - 素材标题输入框
  - 素材类型选择器（理论知识、案例分析、方法论等）
  - 确认上传按钮

### 2. 上传功能实现
- 创建了 App Router API 路由 `/api/file-upload` 处理文件上传
- 实现了文件上传逻辑：
  - 将文件保存到 public/uploads 目录
  - 验证用户认证
  - 根据文件扩展名推断格式
  - 将文件信息保存到数据库

### 3. 页面更新
- 修改了 Raw 素材管理页面，添加了上传对话框触发器
- 实现了数据绑定，显示真实的数据库统计数据：
  - 总文件数
  - 已处理文件数
  - 待处理文件数
  - 成功率计算
- 更新了文件列表，显示从数据库获取的真实文件数据

### 4. 技术细节
- 文件上传通过 Fetch API 发送 FormData 到 /api/file-upload
- 上传成功后刷新页面以显示最新数据
- 文件上传进度和状态反馈
- 文件验证和错误处理

## 使用方式
1. 点击 Raw 素材管理页面上的 "上传素材" 按钮
2. 选择要上传的文件（支持 PDF, Word, Excel, PPT, TXT, MD）
3. 输入素材标题（如未填写，自动使用文件名）
4. 选择素材类型
5. 点击 "确认上传"

## 文件存储
- 上传的文件将存储在 `public/uploads/` 目录
- 文件记录将保存到 PostgreSQL 数据库的 `Raw` 表中
- 文件 URL 将保存为 `/uploads/{filename}` 格式
